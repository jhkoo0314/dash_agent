# [Master Prompt: SFE Intel V13 - Outcome & Risk Integration]

### 1. 역할 및 배경 (Role & Context)

당신은 제약 SFE(영업 효율성) 컨설턴트이자 시니어 풀스택 개발자입니다. 15년 경력의 영업 전문가가 현장에서 즉각적인 의사결정을 내릴 수 있도록, **성과지수(PI)**와 **성장지수(FGR)**를 중심으로 한 대시보드 리빌딩을 수행하십시오. [cite: 2025-10-17, 2026-02-22]

### 2. 상단 6개 카드 리빌딩 전략 (Dynamic Metric Mapping)

담당자 상세 뷰에서 이미 Layer 1에 표시된 [달성률, 효율, 순위, 소속] 정보와의 중복을 피하고, **'결과(Outcome)'**와 **'행동(Behavior)'**을 동시에 모니터링할 수 있도록 설계하십시오.

| **순번**   | **전체/지점 뷰 (Manager Mode)**       | **담당자 상세 뷰 (Coach Mode)**        |
| ---------------- | ------------------------------------------- | -------------------------------------------- |
| **카드 1** | **YTD 누적 달성률**[cite: 2026-02-22] | **성과지수 (PI)**[cite: 2026-02-22]    |
| **카드 2** | **핵심 성공 요인 (CSF)**              | **성장지수 (FGR)**[cite: 2026-02-22]   |
| **카드 3** | **성과지수 (PI)**                     | **유효행동 (HIR)**[cite: 2026-02-22]   |
| **카드 4** | **성장지수 (FGR)**                    | **관계온도 (RTR)**[cite: 2026-02-22]   |
| **카드 5** | **영업 효율성 (M/월)**                | **행동규칙성 (BCR)**[cite: 2026-02-22] |

### 3. 지니 계수 및 커버리지 통합 분석 (Risk & Dominance)

Layer 3의 리스크 분석 섹션을 고도화하십시오. 지니 계수의 정반대 개념인 **'거래처 커버리지'**를 동일 섹션에 배치하여 리스크와 지배력을 동시에 보여주십시오. [cite: 2026-02-22]

* **계산 로직:** **$\text{Coverage} = 1 - \text{Gini}$** [cite: 2026-02-22]
* **시각화:** - 지니 계수는 리스크 게이지(Stable/Warning/Danger)로 표시하십시오.
  * 커버리지는 게이지 하단 또는 측면에 "거래처 장악력: XX%" 형태로 명시하여 숫자가 높을수록 긍정적인 신호임을 강조하십시오. [cite: 2026-02-22]

### 4. 시간 지능 및 데이터 탐지 (Time-Intelligence)

* **기준월 자동 설정:** `monthly_actual` 데이터가 존재하는 마지막 인덱스를 `currentMonth`로 설정하고, 헤더에 "기준: 2026.XX (YTD)"를 명시하십시오. [cite: 2026-02-22]
* **YTD 달성률:** 상단 카드의 달성률은 `currentMonth`까지의 누적 실적과 누적 목표를 합산하여 계산하십시오. [cite: 2026-02-22]

### 5. 기술 구현 상세 (Technical Specs)

* **단위 변환:** `k`는 '만 원', `M`은 '억 원' 단위로 사용자가 이해하기 쉽게 변환하거나 병기하는 함수를 포함하십시오. [cite: 2025-10-17, 2026-02-22]
* **스타일링:** PI나 FGR이 지점 평균보다 낮을 경우 `text-rose-500` 클래스를 동적으로 부여하십시오. [cite: 2026-02-22]
* **차트 연동:** `update()` 함수 호출 시 상단 6개 카드의 라벨과 값이 즉시 변경되도록 `updateMetrics()` 로직을 최우선 실행하십시오. [cite: 2026-02-22]

**이 지시사항을 바탕으로, PI/FGR이 강화된 상단 카드와 지니-커버리지 통합 섹션이 적용된 전체 `<script>` 및 관련 HTML 섹션을 재작성하십시오.**



**[Master Prompt Add-on: Strategic Currency Formatting]**

**1. 지표 명칭 및 단위 현행화:**

* 모든 `k` 단위 지표는 **'만 원'**으로, `M` 단위 지표는 **'억 원'**으로 환산하여 표시하십시오 [cite: 2025-10-17].
* **개인 지표:** '전환 효율' 라벨을 **'점당 매출 기여도(만 원/점)'**로 변경하고, 값에서 `k`를 제거한 후 한국 내 통화 관념에 맞게 환산하십시오 (예: **$35,000k \rightarrow 3,500\text{만 원}$**).
* **지점 지표:** '기간 평균 영업 효율성' 라벨을 **'월평균 처방 실적(억 원)'**으로 변경하고, `M` 단위를 `억` 단위로 변환하십시오 (예: **$8,500M \rightarrow 85.0\text{억 원}$**).

**2. 시각적 가독성:**

* 숫자가 커지는 만큼 천 단위 구분 쉼표(`,`)를 반드시 적용하여 영업 관리자가 즉각적으로 금액 규모를 파악하게 하십시오.
