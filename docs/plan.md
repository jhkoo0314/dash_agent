### [Phase 1] UI/UX 레이아웃 개편 (결과 템플릿 중심)

사용자가 데이터를 탐색하는 흐름을 '성과 확인 -> 원인 분석 -> 처방 도출'의 3단계 계층 구조(Hierarchy)로 재배치합니다.

1. **지점 레벨 (3-Layer View):**
   * **Layer 1 (Performance):** 달성률(`v_achieve`)과 6대 역량 레이더 차트를 나란히 배치하여 지점의 현재 위상을 진단합니다.
   * **Layer 2 (HIR Causal Engine):** **[핵심]** 기존 SHAP 그래프를 삭제하고, **HIR 8대 행동별 실적 기여도 토네이도 차트**를 배치합니다.
   * **Layer 3 (Strategy & Risk):** 시차 보정 매트릭스와 지점별 리스크 데이터를 배치하여 다음 달 전략을 수립하게 합니다.
2. **담당자 레벨 (Coaching View):**
   * **Layer 1 (Individual Status):** 개인 달성률 및 효율성 카드를 배치합니다.
   * **Layer 2 (Behavior Analysis):** 지점 평균 대비 4대 지표(HIR, RTR, BCR, PHR) 격차를 분석합니다.
   * **Layer 3 (Action Prescription):** `Coaching_Rules` 시트의 로직에 따라 개인별 **'맞춤형 영업 처방'** 메시지를 텍스트로 노출합니다.

### [Phase 2] 데이터 엔진 리팩토링 (계산 로직 중심)

새로운 템플릿에 '진짜 데이터'를 공급하기 위해 `report_builder_v12.py`를 전면 개조합니다.

1. **HIR 지표의 원자적 분해:** HIR을 하나의 덩어리로 계산하지 않고, 8대 행동(PT, 시연, 클로징 등)을 개별 변수로 분리하여 분석 데이터프레임(`X`)을 재구성합니다.
2. **지표 연산의 정석화:** RTR(시간 감쇠), BCR(방문 주기 표준편차), Gini(실적 쏠림) 등 모든 지표를 마스터 로직 수식대로 구현합니다.
3. **T-Score 시스템 연동:** `System_Setup` 시트의 평균(70), 편차(10)를 참조하여 모든 지표의 체급을 동일하게 맞춥니다.
4. **인위적 조작 제거:** 상관계수 강제 합산(+0.45) 및 랜덤 데이터 생성을 모두 삭제합니다.
