# Map Security Governance

## 목적
지도 데이터 처리 시 보안/권한/감사 기준을 정의한다.

## 데이터 보호
- 원본(`data/`) 무결성 보장
- 산출물(`output/`) 보존 정책 준수
- 무단 삭제/초기화 금지

## 접근 권한
- 최소 권한 원칙 적용
- 운영자/검토자/승인자 권한 분리
- 정책 변경은 승인 절차 필수

## 감사/추적
- run_id, 실행자, 시각, 시나리오 기록
- 변경 이력(무엇/누가/왜) 기록
- 실패 원인/조치 이력 보관

## 민감정보 처리
- 반출 시 필요한 필드만 최소 제공
- 식별 가능 정보는 마스킹 정책 적용

## 인코딩/문자 정책
- UTF-8 고정
- `PYTHONIOENCODING=utf-8` 설정 후 검증
