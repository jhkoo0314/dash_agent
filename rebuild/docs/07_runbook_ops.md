# 07 Runbook Ops

## 목적
운영 절차(업로드, 실행, 로그 확인, 재실행, 배포)와 장애 대응 플로우를 표준화한다.

## 일상 운영 절차
1. 파일 업로드: 도메인별 입력 반입
2. 시나리오 선택: monthly/quarterly/business
3. 실행: 파이프라인 트리거
4. 로그 확인: 단계별 성공/실패 확인
5. 결과 검토: 핵심 지표 sanity check
6. 배포: HTML 결과 전달

## 재실행 기준
- 매핑 누락/오류 수정 후 재실행
- 키 포맷 문제 수정 후 재실행
- 동일 config로 재현 가능 여부 확인

## 장애 대응 플로우
1. 실패 stage 확인
2. 원인 분류(입력/매핑/키/지표/렌더)
3. 조치 수행
4. 동일 조건 재실행
5. 장애 리포트 기록

## 운영 체크포인트
- 원본 데이터 삭제 금지 (`data/`, `output/`)
- 출력명 규칙 준수(날짜 + 충돌회피 suffix)
- UTF-8 인코딩 준수
