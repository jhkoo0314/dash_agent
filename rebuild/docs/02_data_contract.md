# 02 Data Contract

## 목적
표준 컬럼, 필수 ID, 타입/포맷, 결측/중복 정책을 명문화한다.

## 표준 컬럼 규칙
- 컬럼명은 영문 `snake_case`
- 설명성 컬럼(이름/라벨)은 별도 속성으로 유지
- 표준 컬럼 우선, 원천 컬럼은 매핑으로 흡수

## 필수 키
- `hospital_id`
- `product_id`
- `branch_id` (필요 시)
- 시간키: `metric_month` (`YYYY-MM`)

## 타입/포맷
- ID: string
- 금액/수치: numeric
- 날짜 파생 월: `YYYY-MM`
- 텍스트: UTF-8

## 결측/중복 정책
- 필수 키 결측: 허용치 초과 시 실패
- 조인키 중복: 기본 불허
- 지표 결측: 시나리오 기준 허용치 적용

## 계약 위반 처리
- 위반 단계 즉시 중단(Fail-Fast)
- 로그에 위반 컬럼, 건수, 샘플 기록
