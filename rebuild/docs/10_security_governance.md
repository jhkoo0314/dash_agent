# 10 Security Governance

## 목적
데이터 보호, 권한, 감사, 인코딩 준수 등 운영 거버넌스를 정의한다.

## 데이터 보호 정책
- `data/` 원본 무결성 보장
- `output/` 산출물 보존 정책 적용
- 무단 삭제/초기화 금지

## 접근 권한
- 역할 기반 최소 권한 원칙
- 운영자/검토자/승인자 권한 분리
- 설정 변경은 승인 절차 필수

## 감사/추적
- 실행 로그(run_id, 사용자, 시각, 시나리오)
- 변경 로그(무엇, 누가, 왜)
- 실패 원인 및 조치 이력 보관

## 인코딩/문자 정책
- UTF-8 고정
- 한글 깨짐 출력은 수정 근거로 사용 금지
- 검증 전 `PYTHONIOENCODING=utf-8` 설정

## 보안 운영 수칙
- 민감정보 마스킹 원칙 적용
- 외부 반출 파일 최소화
- 배포 전 품질/보안 체크리스트 통과 필수
