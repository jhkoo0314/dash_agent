# PRD (Product Requirements Document) - 업무자동화 재구축

## 1. 제품 개요
- 제품명: SFE 업무자동화 Rebuild
- 제품 형태: Streamlit 작업 콘솔 + HTML 보고서 생성 엔진
- 목적: 데이터 소스/요구사항 변동이 잦은 환경에서 재현 가능한 보고 자동화 제공

## 2. 문제 정의
현재/기존 운영에서 발생하는 핵심 문제:
- 소스 컬럼 변경 시 코드 수정 비용 증가
- 시나리오 추가 시 엔진 복제 위험
- 조인 오류(중복/누락/행 폭증)로 결과 신뢰도 저하
- 실행 이력/검증 로그 부족으로 재현성 저하

## 3. 목표
- 엔진 1개로 다중 시나리오 운영
- config/template 중심 확장 구조 확립
- Fail-Fast 검증으로 조인 품질 보장
- 실행 로그와 품질 메타데이터를 통한 추적성 확보

## 4. 비목표 (Out of Scope)
- 완전 자율 에이전트 의사결정
- Streamlit 내 복잡 집계 로직 직접 구현
- 데이터 원본 삭제/초기화 자동화

## 5. 사용자
- 1차 사용자: 데이터/영업기획 실무자
- 2차 사용자: 운영/분석 담당자
- 관리자: 매핑/시나리오 정책 승인 담당

## 6. 핵심 사용자 시나리오
1. 사용자가 도메인 파일(sales/targets/crm 등)을 업로드한다.
2. 월간/분기/업무 시나리오를 선택한다.
3. 시스템이 config를 로드하고 엔진을 실행한다.
4. 실패 시 원인 로그를 확인하고 수정 후 재실행한다.
5. 성공 시 HTML 결과물을 다운로드/배포한다.

## 7. 기능 요구사항 (Functional Requirements)
- FR-1: 시나리오 선택 기반 실행
- FR-2: 컬럼 매핑 자동 표준화 (`config/mapping.json`)
- FR-3: 키/그레인 검증 (null/중복/행증가율)
- FR-4: 지표 계산 및 payload 생성
- FR-5: 템플릿 렌더링 기반 HTML 출력
- FR-6: 실행 로그/품질 메타데이터 저장
- FR-7: 날짜 기반 출력명 + 충돌회피 suffix

## 8. 비기능 요구사항 (Non-Functional Requirements)
- NFR-1: 재현성 - 동일 입력+동일 config면 동일 결과
- NFR-2: 확장성 - 시나리오/도메인 추가 시 엔진 수정 최소화
- NFR-3: 안정성 - 검증 실패 시 즉시 중단(Fail-Fast)
- NFR-4: 추적성 - 단계별 로그 및 품질 지표 보관
- NFR-5: 호환성 - mapping 하위호환 유지
- NFR-6: 인코딩 - UTF-8 고정

## 9. 데이터 요구사항
- 표준 컬럼 체계 사용 (snake_case)
- 필수 ID 키 예: `hospital_id`, `product_id`, `branch_id`
- 시간키 포맷: `YYYY-MM`
- Long 우선 표준화, 필요 시 보고 단계에서 Wide 변환

## 10. 아키텍처 요구사항
- 공통 엔진 책임:
  - load -> mapping -> keys/join -> metrics -> payload -> render
- Streamlit 책임:
  - 업로드, 실행 트리거, 로그 확인, 결과 다운로드
- 템플릿 책임:
  - 표현 계층만 담당 (처리 로직 금지)

## 11. 성공 지표 (KPI)
- 신규 시나리오 추가 리드타임
- 수동 수정 없는 성공 실행률
- 조인 오류 재발률
- 매핑 사전 재사용률
- 결과 생성 총 소요시간

## 12. 릴리즈/도입 계획
- Phase 1 (1~2주): 표준 컬럼/ID/시나리오 초안 확정
- Phase 2 (1~2주): 엔진 모듈 분리 + 검증 게이트 구현
- Phase 3 (1주): 시나리오 확장 + 템플릿 정리 + UI 연동
- Phase 4 (지속): 운영 모니터링/품질 리포트/사전 관리 체계화

## 13. 리스크 및 대응
- 리스크: 이름 기반 조인 사용
  - 대응: ID 기반 조인 정책 강제
- 리스크: grain 불일치
  - 대응: 조인 전 grain 검증 의무화
- 리스크: 매핑 변경으로 하위호환 깨짐
  - 대응: 변경 승인 절차 + 이력 관리
- 리스크: 한글 인코딩 깨짐
  - 대응: UTF-8 고정 + 검증 규칙 적용

## 14. 승인 기준
- Streamlit 실행 가능
- Report 빌드 성공 및 output 파일 생성
- 품질 검증 로그 저장 확인
- mapping 하위호환성 유지 확인
